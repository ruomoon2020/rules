# Service-to-Service Authentication

用于内部 HTTP/RPC、MQ producer/consumer、Job、Webhook、调度平台等机器身份与服务间授权。

## 基本原则

1. 禁止仅凭“内网 IP”“调用方说自己是谁”判断可信。
2. 服务间调用必须有机器身份：mTLS、服务网格身份、网关签发 token、HMAC 签名或等价机制。
3. 服务账号须最小权限、分环境、分系统；禁止多个系统共用同一高权限 AK/SK。
4. 机器身份凭证须可轮换、可吊销、可审计，密钥管理见 `36-crypto-key-management.md`。

## 内部调用

1. 内部 API 也须鉴权与限流，尤其是管理、批量、导出、数据修复、任务触发接口。
2. 调用方身份、目标服务、权限、traceId、结果须记录到日志或审计（按敏感级别）。
3. 服务间授权不得绕过租户 / 数据权限边界。

## Webhook / 回调

1. 入站 Webhook 必须校验签名、时间戳、nonce / eventId，防重放。
2. 回调处理须幂等；同一 eventId 重放不得重复扣款、重复发货、重复授权。
3. Webhook secret 分合作方管理，支持轮换和失效。

## MQ / Job 身份

1. MQ consumer、定时任务、批处理必须有明确运行身份与权限范围。
2. 禁止任务以超级管理员身份无边界访问所有租户数据；确需跨租户须审计和原因。

## AI 生成约束

1. AI 不得把内部接口默认设为无认证。
2. AI 生成 Webhook / MQ / Job 代码时，必须考虑签名、幂等、重放防护和审计。
