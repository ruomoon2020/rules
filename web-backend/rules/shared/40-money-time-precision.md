# Money, Time, and Precision

金额、时间、时区、精度是企业后台的高频事故源，须在领域模型、DTO、数据库和契约中统一。

## 金额

1. 禁止使用 `double` / `float` 表示金额、费率、余额、积分兑换比例。
2. 金额使用 `BigDecimal` 或最小货币单位 `long`；团队须统一一种主策略。
3. 金额字段必须明确币种、精度、舍入规则；跨币种禁止直接相加。
4. 数据库金额列须定义精度与 scale；禁止无说明使用 `decimal(10,2)` 套所有场景。
5. 金额计算须有边界测试：0、负数、超大值、舍入、税费、折扣。

## 时间与时区

1. 存储默认 UTC 或项目统一时区；接口字段须明确格式和时区。
2. 推荐使用 `Instant` / `OffsetDateTime` / `LocalDate` 按语义区分时刻、带偏移时间、自然日。
3. 禁止用服务器本地时区隐式决定跨地区业务逻辑。
4. 自然日、账期、结算日、活动截止时间必须说明时区和闭开区间。
5. 定时任务跨时区 / 夏令时场景须有测试或说明。

## AI 生成约束

1. AI 看到金额字段不得使用 `double` / `float`。
2. AI 生成时间字段、查询区间、定时任务时，必须说明时区、边界和格式。
