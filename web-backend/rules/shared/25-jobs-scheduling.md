# Jobs and Scheduling Rules

用于定时任务、批处理、异步任务、补偿任务。

## 基本要求

1. 定时任务必须有明确 owner、触发频率、超时、重试、告警和关闭开关。
2. 分布式部署下任务必须有防重方案：分布式锁、调度平台单实例、分片策略或幂等处理。
3. 任务处理必须可恢复：记录任务状态、游标、批次、失败原因。
4. 批处理必须分批提交，禁止一次事务处理海量数据。
5. 任务日志必须包含 jobName、jobId、traceId、batchNo、durationMs、result。

## 幂等与补偿

1. 重试任务必须幂等，重复执行不得重复扣款、重复发消息、重复变更权限。
2. 外部副作用（发 MQ、发邮件、调用第三方）应有去重键或 outbox / 事件表。
3. 补偿任务必须记录补偿对象、原因、执行结果和人工介入入口。

## 禁止

- 多实例下无锁跑同一个定时任务。
- catch 异常后只打印日志不更新任务状态。
- 在定时任务里绕过权限、租户、审计与数据脱敏。
- 长事务包住远程调用或大批量循环。
