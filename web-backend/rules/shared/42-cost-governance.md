# Cost Governance

成本不是只属于云平台团队。后端代码中的查询、导出、日志、对象存储、第三方调用和批处理都会形成长期成本。

## 触发条件

以下变更须说明成本影响：

1. 新增外部付费服务、短信、邮件、实名、支付、地图、OCR、AI 接口。
2. 新增大导出、大查询、批处理、归档、备份、日志 / 审计长期保留。
3. 新增缓存、MQ、对象存储、搜索引擎、云资源、跨区复制。
4. 提高默认保留周期、日志级别、采样率、导出上限。

## 规则

1. 批处理和定时任务必须有扫描范围、频率、单次上限，禁止无界全表扫描。
2. 导出、查询、第三方调用须有配额或限流，避免被误用产生高额费用。
3. 日志、审计、归档、备份须平衡合规、排障和成本；保留周期写入配置或文档。
4. 付费外部调用须记录调用量、失败率、重试次数，避免失败重试放大成本。
5. 新增云资源须说明 Owner、环境、预估费用、清理策略。

## AI 生成约束

1. AI 不得建议无上限导出、无采样日志、无限重试第三方付费接口。
2. AI 涉及付费资源或高数据量任务时，必须提示成本、配额、Owner 和清理策略。
