# Dictionary and State Machine

用于字典、枚举、状态码、主数据和业务状态流转治理，避免“改一个状态值，多个系统一起坏”。

## 字典 / 枚举

1. 字典码、枚举值、状态码是契约；禁止静默改语义。
2. 新增、废弃、删除字典值须有 Owner、影响范围、前后端展示策略和迁移说明。
3. 对外 API 枚举以 OpenAPI 为准；后端常量 / enum 与契约同步。
4. 展示文案走前端 i18n / 字典服务 / 约定映射，禁止后端返回不可控的 UI 文案替代稳定码。

## 状态机

1. 有业务流转的状态必须定义合法迁移表：from、to、触发动作、权限、前置条件、审计。
2. 禁止任意接口直接 `setStatus` 绕过状态机。
3. 状态变更须考虑并发：乐观锁、幂等键、重复请求语义。
4. 关键状态变更须审计 before / after、operator、reason、traceId。

## 主数据

1. 组织、角色、权限码、地区、产品、渠道等主数据须定义来源系统与同步规则。
2. 禁止在业务模块复制一份长期分叉的主数据表。

## AI 生成约束

1. AI 不得新增魔法状态值或字典值后只改局部代码。
2. AI 生成状态流转代码时，必须说明合法迁移、权限、审计和并发控制。
