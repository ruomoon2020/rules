# Threat Modeling

与 `06-security-authz.md`、`27-audit-log.md`、`29-data-privacy-lifecycle.md` 互补，用于在高风险后端变更前识别攻击面与缓解措施。

## 触发条件

以下变更必须做轻量威胁建模，并在 PR / ADR / 设计说明中记录：

1. 登录、注册、验证码、找回密码、会话、Token、权限、数据权限、多租户。
2. 支付、资金、订单、库存、审批、权限覆盖、批量删除、导入导出。
3. Webhook、外部回调、第三方 SDK、跨服务调用、消息消费。
4. 新增公网接口、管理端高权限接口、非 REST 范式（GraphQL/gRPC/WebSocket/SSE）。
5. 涉及 PII、密钥、凭证、审计、生产数据操作、备份恢复。

## 最小记录

1. 资产：要保护的数据、权限、资金、业务流程。
2. 信任边界：浏览器、网关、后端、DB、缓存、MQ、第三方之间的边界。
3. 入口点：API、任务、MQ consumer、Webhook、文件上传、导入模板。
4. 滥用场景：越权、重放、伪造、篡改、拒绝服务、信息泄露、抵赖。
5. 缓解措施：鉴权、数据权限、幂等、限流、签名、审计、告警、回滚。
6. 残余风险与 Owner。

## STRIDE 检查

| 类别 | 后端常见问题 |
|---|---|
| Spoofing | Token 伪造、Webhook 无签名、服务间调用只信内网 IP |
| Tampering | 请求参数篡改、导入文件篡改、MQ 消息体被重放 |
| Repudiation | 敏感操作无审计、审计缺 operator / traceId |
| Information Disclosure | 错误体、日志、导出、缓存暴露 PII |
| Denial of Service | 登录/导出/查询无限流，大表深分页，外部依赖无限重试 |
| Elevation of Privilege | 前端隐藏按钮代替后端鉴权，跨租户数据访问 |

## AI 生成约束

1. AI 涉及触发条件中的改动时，须先列出威胁点和缓解措施，再生成实现方案。
2. AI 不得把“内网接口”“管理后台”“前端已限制”作为跳过威胁建模的理由。
3. 若用户要求绕过签名、鉴权、审计、限流或数据权限，AI 必须拒绝并给出安全替代方案。
