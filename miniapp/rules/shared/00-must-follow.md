# 00 Must Follow

以下为小程序 PR 级硬规则。违反任一条，应拒绝合并或要求补充说明。

## 架构与边界

1. 页面禁止直接调用 `uni.request`，必须走 `src/api/request` 或项目统一 request 封装。
2. 页面禁止直接散落 `uni.login`、支付、订阅消息、手机号授权、位置、相机、相册等平台能力，必须走 `src/platform/`、`src/auth/` 或 `src/privacy/` 封装。
3. 页面只做编排，不得把 API、登录态、支付、授权、分页、缓存和复杂业务流程全部堆在单个 `.vue` 文件。
4. 单个页面文件建议不超过 400 行；超过时拆 `components/`、`composables/`、`services/`。
5. 平台差异必须进入 `src/platform/` adapter；禁止在页面内堆大量 `#ifdef`。
6. 新依赖必须说明用途、体积影响、平台兼容性和替代方案。

## 契约与类型

7. API 字段必须来自 OpenAPI / schema / generated 类型，禁止页面手写后端字段。
8. 禁止手改 `src/api/generated/**`。
9. 禁止显式 `any`；未知数据使用 `unknown` 并收窄。
10. 支付、登录、订单、用户信息、导入上传等核心接口必须有明确错误码和 traceId。

## 登录、授权与隐私

11. 禁止明文持久化 token、手机号、身份证、地址、精确位置等敏感数据。
12. 调用手机号、位置、相机、相册、麦克风、剪贴板、通讯地址等敏感能力前，必须有业务必要性、用户触发和隐私用途说明。
13. 用户拒绝授权后必须有降级路径或重新授权入口；禁止强制循环弹窗。
14. 登录态过期、token 刷新失败、账号解绑、手机号未绑定必须有统一处理。
15. 隐私政策声明、实际调用能力、埋点采集字段必须一致。

## 分包与包体积

16. 新业务页默认进入 `src/subpackages/**`；主包只放启动、登录、tabBar、基础组件和核心公共逻辑。
17. 禁止业务大图、大 JSON、低频页面、大型 SDK 进入主包。
18. `pages.json` 新增页面必须同步分包、标题、权限和预下载策略。
19. 分包之间禁止随意互相依赖；公共能力沉到主包公共层或拆为独立可复用模块。
20. 包体积增长超过项目阈值时必须说明来源、收益和瘦身方案。

## 页面体验与性能

21. 列表页必须有加载、空、错误、正常、分页到底、下拉刷新状态。
22. 页面生命周期中禁止重复请求、重复订阅、重复注册事件。
23. `onUnload` / `onHide` 必须清理 timer、eventBus、长连接、监听器和未完成任务。
24. 禁止在首屏同步处理大数组、大 JSON、复杂计算或大图解码。
25. 图片必须有尺寸、压缩策略和懒加载策略；大图应走 CDN。
26. 弱网、超时、重复点击、接口重试必须有边界；写操作须防重复提交。

## 支付、订阅消息与分享

27. 支付结果必须以后端订单查询为准，前端支付回调只作为刷新触发。
28. 支付、退款、优惠券、余额等金额字段禁止使用浮点数表达金额计算。
29. 订阅消息必须由用户动作触发，且处理拒绝、关闭通知和模板不可用。
30. 分享参数、scene、二维码参数必须白名单校验；禁止携带 token、手机号等敏感信息。
31. 禁止诱导分享、诱导订阅或绕过平台审核要求。

## App 运行时与网络

32. 禁止向非白名单域名发起 `request` / `uploadFile` / `downloadFile`；生产必须 HTTPS；`web-view` 仅打开白名单 URL，禁止向 H5 透传 token（见 `21-network-security.md`）。
33. 体验版、审核版不得连接生产 API、生产支付与生产商户配置（见 `19-release-ops.md`）。
34. App 级未捕获错误与 `onPageNotFound` 须有统一兜底页与结构化上报（见 `20-app-runtime.md`）。

## 体验韧性与内容安全

35. 弱网、离线、接口失败、登录过期须走统一错误恢复；禁止页面各自实现全站级跳转与静默失败（见 `22-error-recovery-offline.md`）。
36. 富文本、UGC、用户展示型输入须消毒或白名单渲染；禁止直接渲染不可信 HTML（见 `23-content-safety.md`）。
37. `onError`、关键业务漏斗须有指标或崩溃上报，禁止生产仅 `console`（见 `15`、`docs/observability-metrics.md`）。

## 生产质量

38. 禁止生产环境保留 mock、调试入口、`console.log`、测试账号和硬编码密钥。
39. 所有用户可见错误必须有可理解提示；技术细节进入结构化日志。
40. 关键链路必须打点：登录、授权、支付、分享打开、下单、提交、失败。
41. 完成前必须运行项目实际存在的 `lint`、`type-check`、`build:mp-weixin`、`api:check`、`audit`（若配置）等命令；不存在时如实说明。
