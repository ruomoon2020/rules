# 25 Dependency Supply Chain

npm 依赖与第三方 SDK 治理。新依赖说明见 `00` §6。

## 引入前

1. 须说明：用途、体积、平台兼容性、许可证（MIT/Apache 等）、维护状态、替代方案。
2. 禁止引入含不明原生能力、与小程序运行时不兼容的 Web-only 库。
3. 第三方统计 / 埋点 / 地图 / 客服 SDK 须登记域名（`21`）与隐私影响（`09`）。

## CI 与锁文件

1. 使用锁文件（`pnpm-lock.yaml`）；禁止无锁文件发版。
2. PR 建议运行 `pnpm audit` 或组织等价漏洞扫描；高危须修复或书面例外。
3. 禁止依赖 `latest` 浮动版本进生产分支。

## 禁止

- 从不可信源安装依赖；禁止提交 `node_modules` 内手改包。
- 为省事复制整包 SDK 进主包而不评估体积。
