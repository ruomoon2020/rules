# 24 Design System Mobile

移动端与设计系统一致性。页面骨架见 `04`；Base 组件目录见 `01`。

## Token 与样式

1. 颜色、字号、间距、圆角优先使用项目 Token（CSS 变量或 `@/styles/tokens`），禁止页面散落魔法数。
2. 适配使用 `rpx` / 安全区（`safe-area-inset`）；全屏页须处理刘海与底部横条。
3. 暗色模式（若支持）须 Token 切换，禁止硬编码仅浅色。

## 触控与可读性

1. 可点击热区建议不小于 88rpx×88rpx（约 44pt 等价），图标按钮须有可读文案或 `aria-label` 等价说明。
2. 表单错误提示靠近字段；destructive 操作用色与主按钮区分。
3. 适老化 / 系统大字号下布局不得重叠截断（关键页抽测）。

## Base 组件

1. 空态、错误态、骨架屏、加载态使用 `base/` 统一组件，与 `12` 四态一致。
2. 禁止每业务页自创一套样式迥异的按钮/列表空态。
3. 图标字体 / 图片须有占位与加载失败态。

## 禁止

- 复制粘贴大段 inline style 覆盖全局主题。
- 关键流程仅图标无文字且无无障碍说明。
