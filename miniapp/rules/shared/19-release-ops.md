# 19 Release Ops

## 版本与基础库

1. `manifest.json` 须声明最低基础库版本；Breaking 平台 API 变更须评估升级策略。
2. 体验版、审核版、正式版配置分离；禁止把调试 AppId / 密钥提交仓库。
3. 多平台（微信 / 支付宝等）发布顺序与回滚方案写在项目 README 或 `99-project-local.mdc`。

## 环境隔离

1. `dev` / `test` / `staging` / `prod` 的 API baseURL、AppId、支付商户号（`merchantId`）、订阅模板 ID 等**仅通过构建变量或 CI 注入**，禁止在页面 / 组件内 `if (体验版)` 切换生产配置。
2. 体验版、审核版**禁止**连接生产 API、生产支付环境与生产商户号（`00` 硬规则）。
3. 审核版临时开关（演示数据、跳过校验等）必须：默认关闭、可追溯（配置来源 + 操作人）、发版前可验证已关闭。
4. 环境枚举与变量名写在 `99-project-local.mdc` 或业务仓 README，例如 `VITE_APP_ENV`、`VITE_API_BASE_URL`。

## 构建与体积

1. 发版前必须 `build:mp-weixin` 通过；主包体积不得超过项目预算（`10`）。
2. `size:check` 超阈值须失败或经 Owner 审批并记录瘦身计划。
3. 生产构建禁止 mock、console、测试账号、调试入口（`00`）。

## 灰度与开关

1. 功能开关、实验配置走远端配置或后端；禁止硬编码长期开关。
2. 开关默认值须安全（关闭高风险能力）；失败时降级策略明确。

## 审核与合规

1. 类目、支付、UGC、直播等须符合微信平台类目与审核要求；见 `docs/compliance-wechat-checklist.md`。
2. 隐私政策、manifest 声明、实际调用能力、埋点字段一致（`09`）。

## 发布后

1. 关键漏斗监控：启动、登录、授权、下单、支付、分享（`15`）。
2. 线上问题用 traceId 关联后端日志；禁止仅靠用户截图排查。
