# 11 Platform Differences

## Adapter 原则

1. 微信、支付宝、抖音等差异进入 `src/platform/`。
2. 页面调用统一业务方法，例如 `requestPayment`、`requestSubscribeMessage`。
3. adapter 返回统一结果类型，页面不直接判断平台原始错误。
4. 条件编译必须集中、可测试、可替换。

## 平台能力

1. 支付、订阅消息、分享、登录、手机号授权必须先支持微信小程序。
2. 其他小程序平台支持时，必须补充差异说明和 fallback。
3. 平台 API 不可用时不得白屏；必须有不可用提示或隐藏入口。
4. 基础库版本要求必须记录在本地覆盖层或 README。

## 能力矩阵（维护于项目文档或 `99-project-local`）

| 能力 | 微信 | 支付宝 | 抖音 | 备注 |
|---|---|---|---|---|
| 登录 | 必须 | 按需 | 按需 | adapter 统一 |
| 支付 | 必须 | 按需 | 按需 | 以后端订单为准 |
| 订阅消息 | 必须 | 差异大 | 差异大 | 模板 ID 集中配置 |
| 分享 | 必须 | 按需 | 按需 | scene 白名单 |
| 手机号 | 必须 | 按需 | 按需 | 用户触发 + 隐私 |

未支持平台不得生成该平台专属 API 调用；须 fallback 或隐藏入口。

## 禁止

- 禁止把 `#ifdef MP-WEIXIN` 散落在多个页面。
- 禁止页面直接消费平台原始返回结构。
- 禁止为了多端兼容牺牲微信主链路的正确性和合规性。
