# 08 State Storage Cache

## Store

1. Pinia 只保存跨页面状态；页面局部状态留在页面或 composable。
2. auth、user、cart、settings 等 store 必须有清理方法。
3. store 不保存接口原始大对象，保留必要字段。

## Storage

1. storage key 集中管理，禁止散落字符串。
2. storage 数据必须有版本、过期策略或迁移策略。
3. 禁止持久化敏感明文。
4. logout、账号切换、租户切换必须清理相关缓存。

## 数据分级与留存

1. 公开配置、字典、非个人化静态数据：可缓存，但须有版本或 TTL。
2. 用户资料、偏好、地址摘要：仅短期缓存；logout、账号切换、租户 / 组织切换必须清理。
3. 手机号、身份证、详细地址、精确位置、支付凭证：默认不进 storage；确需缓存须有业务必要性、最短 TTL、脱敏或加密方案。
4. token、refreshToken、sessionKey：禁止 URL、日志、埋点、分享参数传递；禁止长期明文持久化。
5. 本地缓存 key 若与用户、租户、组织有关，必须包含隔离维度，禁止跨账号复用。

## Cache

1. 缓存必须说明 key、TTL、失效条件和数据权限边界。
2. 用户维度缓存必须包含用户或租户隔离维度。
3. 接口失败时不得用过期缓存伪装为最新成功，除非 UI 明确提示。
