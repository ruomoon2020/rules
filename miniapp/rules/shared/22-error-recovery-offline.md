# 22 Error Recovery Offline

页面级与全站级错误恢复、弱网与离线体验。App 级全局错误见 `20-app-runtime.md`；列表四态见 `12-list-form-pagination.md`。

## 统一入口

1. 登录过期、token 失效、租户切换、权限不足、维护降级必须走 `auth/` 或统一 recovery 服务，禁止页面散写跳转逻辑。
2. 分包加载失败、页面 JS 异常须有统一提示：返回首页、重试、联系客服（按产品）。
3. 关键操作失败不得静默；须保留用户输入（表单草稿）或可重试。

## 401 / 403 / 无权限

1. `401`、token 过期、登录态失效：进入统一登录恢复流程；恢复后按来源页继续或回到安全首页。
2. `403`、`NO_PERMISSION`、数据范围不足：展示无权限提示、隐藏或禁用对应入口；**不得简单跳登录制造循环**。
3. 后端返回业务无权限时，前端不得伪造成功或只靠隐藏按钮；须保留 traceId 便于排查。
4. B 端小程序权限变更后，应刷新后端权限快照或重新拉取用户能力；C 端小程序不应因此引入完整 RBAC 壳层。

## 弱网与离线

1. 须识别无网 / 弱网（`uni.onNetworkStatusChange` 或等价），展示统一 offline 提示，禁止无限 loading。
2. 读请求失败：页面级重试；写请求失败：明确提示，禁止自动无限重试。
3. 禁止用过期缓存伪装「最新成功」除非 UI 标明「离线数据」（交叉 `08`）。
4. 轮询、支付状态查询在弱网下须有退避与上限。

## 页面级恢复

1. 列表 / 详情 / 表单 error 态须提供「重试」或返回路径，不得仅 toast 一句「失败」。
2. 支付中断、授权取消、订阅拒绝须可恢复业务路径（重新支付、重新授权入口）。
3. 首屏超时建议监控并引导刷新（指标见 `docs/observability-metrics.md`）。

## 禁止

- 捕获错误后空 `catch` 或仅 `console`。
- 每页复制一套 `uni.reLaunch` 登录逻辑。
- 白屏无兜底、无 traceId 对用户不可查。
