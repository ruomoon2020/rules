# 10 Performance Package Size

## 包体积

1. 主包体积必须有预算；超过预算要先瘦身或分包。
2. 分包体积也必须有预算；单个分包超阈值须拆分或说明收益。
3. 大图片、大 JSON、低频 SDK 不得进入主包。
4. 依赖引入要评估 tree-shaking、平台兼容和产物体积。
5. 静态资源优先 CDN；主包只放必要小图标和启动必需资源。
6. 禁止 base64 大图、活动大图、整包图标库进入主包。
7. 定期使用小程序开发者工具或等价工具分析代码依赖。

## 首屏性能

1. 首屏接口按必要性分级：必需、可延后、用户触发；非关键数据延后加载。
2. 禁止首屏并发拉全量字典、全量配置、低频业务数据。
3. 首屏失败应局部降级或展示可恢复错误态，禁止整页白屏。
4. 首屏图片使用压缩、尺寸声明和懒加载。
5. 骨架屏 / 占位态必须和真实布局接近，避免跳动。
6. 禁止首屏同步解析大 JSON 或执行复杂计算。

## 图片与静态资源

1. 列表图片必须使用缩略图；详情页按需加载原图。
2. 图片必须声明宽高或使用稳定容器，避免布局抖动。
3. 非首屏图片必须懒加载；活动图、背景图、大 banner 应走 CDN。
4. 用户上传图片预览必须使用压缩图或服务端缩略图。
5. 新增静态资源 PR 须列出资源大小与所在包（主包 / 分包 / CDN）。

## 运行性能

1. 列表分页加载，禁止一次渲染大量数据。
2. 长列表须虚拟列表、`recycle-view` 或平台推荐方案；禁止单次 setData 超大数组导致卡顿。
3. 控制 WXML 节点数量；避免列表项内嵌套过深组件。
4. 高频输入和滚动事件必须防抖、节流或避免绑定重逻辑。
5. 避免深层响应式大对象、深 watch 和模板内复杂计算。
6. 列表追加只追加新增页，禁止每次加载都重建全量数组。
7. `onShow` 不得无条件重置整页状态导致闪烁或重复请求。
8. 离页后请求结果不得更新已销毁页面。
9. 轮询必须有停止条件、退避策略和可见性控制。

## 分包预加载

1. `preloadRule` 只用于高概率下一跳分包。
2. 禁止一次预加载多个低频分包。
3. 预加载失败不得影响当前主流程。
4. 预加载收益应通过指标验证，长期无收益须移除。
