# 国际化与区域格式规则

1. 用户可见文案、状态展示、错误码映射必须走项目 i18n / 字典服务；禁止在页面复制稳定业务文案或枚举语义。
2. 金额、数量、日期、时间和时区使用统一 formatter（如项目封装的 `Intl`）；接口传输格式与展示格式分离。
3. 不得把浏览器 locale、系统时区或时钟当作业务事实；账期、截止时间、跨时区筛选须明确后端时区和闭开区间。
4. 新增枚举、后端 `errorCode` 或未知 locale 必须有 fallback；不能因缺翻译导致白屏或展示原始异常。
5. 语言切换后，刷新依赖 locale 的字典、格式化缓存和当前页面标题；不得刷新登录态或丢失未提交表单。
