# 错误恢复与白屏治理规则

用于全局异常、路由失败、白屏恢复、接口错误与用户可恢复体验。

## 全局错误

1. 应有全局 error handler，捕获 Vue error、未处理 Promise rejection、资源加载错误。
2. 错误上报字段遵守 `18-logging-observability.md`：`event`、`route`、`traceId`、`release` 等。
3. 错误提示不得暴露堆栈、Token、接口完整响应或敏感字段。
4. 全局错误页必须提供返回首页、刷新、联系支持或重试入口。

## 路由与 Chunk 加载失败

1. 路由懒加载 chunk 失败须有重试或刷新提示，不得永久白屏。
2. 新版本发布导致旧 chunk 404 时，应提示刷新页面或自动软刷新（按项目策略）。
3. 权限失效、登录过期、租户切换失效必须走统一处理，不在页面散写。

## 页面级恢复

1. 页面必须区分 loading、empty、error、permission 状态。
2. 列表、详情、表单提交失败须提供可理解错误与重试路径。
3. 关键操作失败后不得丢失用户输入；表单提交失败保留草稿。
4. 文件导入导出失败须能查看任务状态、失败原因、重试或下载错误明细。

## 白屏检测

1. 生产建议接入白屏检测或首屏超时监控。
2. 白屏监控须带 release、route、用户环境、资源错误摘要。
3. 白屏恢复策略须避免无限刷新循环。

## 禁止

- 捕获错误后静默吞掉。
- 只显示“系统异常”且无 traceId / 重试 / 支持路径。
- 在页面组件里重复实现登录过期、权限失效、全局 500 处理。
