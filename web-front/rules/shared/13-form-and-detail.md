# Form and Detail Page Rules

表单页、详情页的补充约束（列表页见 `04-ui-patterns.md`）。

## 表单

1. 字段来自 schema / generated，不凭空增加。
2. 必填、长度、格式、枚举与契约一致。
3. 异步校验须有 pending / loading 反馈。
4. 提交失败保留用户输入。
5. 只读详情勿用可编辑表单样式，避免误解。
6. 大表单优先 Drawer 或分步，避免小弹窗塞满字段。

## 详情页

1. 结构建议：页头 + 描述列表（Descriptions）+ 操作区 + 关联 Tab（若有）。
2. 展示字段与 schema 一致；敏感字段脱敏。
3. 时间、金额、状态走 `utils/format` 与统一状态标签。
4. 审计日志、操作记录按模块需求展示，不阻塞主信息。

## 字段状态

- 禁用须表达原因（tooltip / 文案）。
- 依赖字段联动时，清除无效联动值。

执行以本文件、`12-schema-ssot.md` 与业务仓现有 Base 表单 / 详情组件为准。
